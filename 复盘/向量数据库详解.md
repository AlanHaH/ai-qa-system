# 向量数据库详解

## 一、什么是向量数据库？

### 普通数据库 vs 向量数据库

**普通数据库（MySQL）**：存结构化数据（文字、数字、日期）
```
| id | name  | age |
|----|-------|-----|
| 1  | 张三  | 20  |
| 2  | 李四  | 22  |
```
查询方式：`SELECT * FROM users WHERE name = '张三'`（精确匹配）

**向量数据库（ChromaDB）**：存向量 + 原始文本
```
| id           | document           | embedding              |
|--------------|--------------------|------------------------|
| doc_1_chunk_0| MySQL是关系型数据库 | [0.1, -0.3, 0.5, ...] |
| doc_1_chunk_1| 创建表用CREATE TABLE| [0.2, -0.1, 0.4, ...] |
```
查询方式：`collection.query(query_texts=["怎么建表"])`（语义匹配）

### 为什么需要向量数据库？

```
用户问："MySQL 怎么建表？"

普通数据库：
  → 搜索关键词 "建表"，找不到就不返回
  → 如果文档写的是 "创建表"，就搜不到

向量数据库：
  → "MySQL 怎么建表" 和 "创建表的语法" 意思相近
  → 向量相似度高，能检索到相关片段
```

## 二、什么是向量？

向量就是一个**浮点数数组**，用来表示文本的**语义**（意思）。

```
"MySQL 怎么建表"    → [0.1, -0.3, 0.5, 0.2, ...]
"创建数据库表的方法" → [0.12, -0.28, 0.48, 0.22, ...]  ← 很相似！
"今天天气真好"      → [0.9, 0.1, -0.7, 0.3, ...]       ← 完全不相关
```

### 向量相似度计算

常用**余弦相似度**（cosine similarity）：
- 值越接近 **1**，越相似
- 值越接近 **0**，越不相关

```
"MySQL 建表" 和 "创建数据库表" → 相似度 0.95（很相似）
"MySQL 建表" 和 "今天吃什么"   → 相似度 0.12（不相关）
```

## 三、ChromaDB 用法

### 1. 安装

```powershell
pip install chromadb
```

### 2. 初始化

```python
import chromadb

# 创建客户端
# path="chroma_db" — 数据存在这个目录，重启后还在
# PersistentClient — 持久化存储
chroma_client = chromadb.PersistentClient(path="chroma_db")

# 获取或创建集合（类似 MySQL 的"表"）
# 如果 "documents" 不存在就创建，存在就获取
collection = chroma_client.get_or_create_collection(name="documents")
```

### 3. 添加数据

```python
collection.add(
    ids=["doc_1_chunk_0"],                              # 唯一标识
    documents=["MySQL 是一个关系型数据库管理系统..."]      # 原始文本
)
# ChromaDB 会自动把文本转换成向量，不需要手动调 Embedding API
```

**批量添加**：
```python
collection.add(
    ids=["doc_1_chunk_0", "doc_1_chunk_1", "doc_1_chunk_2"],
    documents=["第一段内容...", "第二段内容...", "第三段内容..."]
)
```

### 4. 查询数据

```python
# 语义搜索：找和问题最相似的文档片段
results = collection.query(
    query_texts=["MySQL 怎么建表"],  # 用户的问题
    n_results=3                       # 返回最相似的前3个
)

print(results)
# {
#   "ids": [["doc_1_chunk_1", "doc_1_chunk_0", "doc_1_chunk_2"]],
#   "documents": [["创建表的语法是...", "MySQL 是...", "表由列组成..."]],
#   "distances": [[0.15, 0.23, 0.31]]
# }
```

**获取结果**：
```python
# 只取文档内容
docs = results["documents"][0]  # ["创建表的语法是...", "MySQL 是...", ...]

# 只取距离（越小越相似）
distances = results["distances"][0]  # [0.15, 0.23, 0.31]
```

### 5. 查看所有数据

```python
# 获取所有数据
data = collection.get()
print(data)
# {
#   "ids": ["doc_1_chunk_0", "doc_1_chunk_1", ...],
#   "documents": ["MySQL 是...", "创建表...", ...],
#   "embeddings": [[0.1, -0.3, ...], [0.2, -0.1, ...], ...]
# }

# 查看总数
count = collection.count()
print(f"共 {count} 条数据")

# 查看前 N 条
peek_data = collection.peek(3)
```

### 6. 删除数据

```python
# 删除指定数据
collection.delete(ids=["doc_1_chunk_0"])

# 删除所有数据
collection.delete(where={})

# 删除整个集合
chroma_client.delete_collection("documents")
```

### 7. 更新数据

```python
# 更新指定数据
collection.update(
    ids=["doc_1_chunk_0"],
    documents=["新的内容..."]
)
```

## 四、ChromaDB 常用操作速查表

| 操作 | 代码 |
|------|------|

| 删除所有 | `collection.delete(where={})` |
| 更新数据 | `collection.update(ids=[...], documents=[...])` |

## 五、本项目中的用法

### 上传文档时存入向量库

```python
# services/rag_service.py

def add_document(doc_id: int, text: str):
    """把文档切分后存入向量库"""
    chunks = split_text(text)  # 切分成小块
    for i, chunk in enumerate(chunks):
        collection.add(
            ids=[f"doc_{doc_id}_chunk_{i}"],  # 唯一标识：doc_1_chunk_0
            documents=[chunk]                   # 原始文本
        )
```

### RAG 问答时检索

```python
# services/rag_service.py

def search_similar(question: str, top_k: int = 3) -> list:
    """根据问题检索最相关的文档片段"""
    results = collection.query(
        query_texts=[question],  # 用户的问题
        n_results=top_k          # 返回前3个
    )
    return results["documents"][0] if results["documents"] else []
```

### 查看向量库数据

```python
# 查看所有片段
data = collection.get()
for i in range(len(data["ids"])):
    print(f"ID: {data['ids'][i]}")
    print(f"内容: {data['documents'][i][:100]}...")
    print("---")
```

## 六、向量数据库 vs MySQL 对比

| 对比项 | MySQL | ChromaDB |
|--------|-------|----------|
| 存什么 | 结构化数据（文字、数字） | 向量 + 原始文本 |
| 怎么查 | SQL 语句，精确匹配 | 语义相似度检索 |
| 适合场景 | 用户信息、订单、聊天记录 | 文档检索、问答系统 |
| 查询示例 | `SELECT * FROM users WHERE name='张三'` | `collection.query(query_texts=["张三是谁"])` |
| 数据结构 | 行和列（二维表） | id + document + embedding |
| 索引方式 | B+ 树索引 | 向量索引（HNSW 等） |

## 八、常见问题

### ChromaDB 数据存在哪？
存在 `backend/chroma_db/` 目录里，重启后数据还在。

### 怎么清空向量库？
```python
import chromadb
client = chromadb.PersistentClient(path="chroma_db")
collection = client.get_collection("documents")
collection.delete(where={})
```

### 怎么删除整个向量库？
直接删除 `backend/chroma_db/` 目录即可。

### ChromaDB 和 FAISS 的区别？
- ChromaDB：功能完整，自带持久化、Embedding，适合中小项目
- FAISS：Facebook 开源，性能更好，适合大规模数据
- 本项目用 ChromaDB，因为它更简单易用

### 向量维度是什么？
向量的长度（数组里有多少个数）。维度越高，能表示的语义越细，但计算越慢。ChromaDB 默认使用 384 维。
